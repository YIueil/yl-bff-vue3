# 图标使用指南组件计划

## 目标与范围

为项目增加一个可从现有 `/icon` 路由直接访问的图标使用指南组件，完整说明并现场演示项目支持的所有图标使用方式：

- `@iconify/vue` 在线图标。
- `unplugin-icons` 构建期按需生成的离线图标组件。
- 通过本地 Iconify JSON 注册的自定义单色图标。
- 通过本地 Iconify JSON 注册的自定义多色图标。
- 图标尺寸、颜色和动态图标名称等常见用法。
- 自定义 SVG 的目录约定、JSON 生成命令和应用注册流程。

本任务只完善图标说明与演示，不修改 Iconify 集成架构，不新增图标库、样式库或代码高亮依赖。

## 现状问题

- `src/views/IconView.vue` 只连续渲染若干图标，缺少标题、分组、代码示例和使用边界说明。
- 在线图标、构建期离线图标、本地自定义图标之间的加载方式和适用场景不明确。
- 页面没有说明自定义单色图标支持 `currentColor`，而多色图标的内置颜色通常不会被 `color` 覆盖。
- 页面没有展示 `pnpm run icons:generate`、SVG 输入目录、JSON 输出目录和启动注册文件之间的关系。
- 示例没有解释动态图标名称只适用于 `<Icon>` 形式，构建期组件名需要静态可分析。

## 推荐方案

### 组件结构

新增 `src/components/IconUsageGuide.vue`，并在现有 `src/views/IconView.vue` 中显式导入和渲染。保留 `/icon` 路由及导航不变，避免为了说明页增加新的公共入口。

指南组件使用语义化 HTML 和 scoped CSS，包含：

1. 页面标题和用途简介。
2. 使用方式对比区，说明来源、是否依赖网络、写法和适用场景。
3. 在线 Iconify 区：
   - 显式导入 `Icon`。
   - 使用 `mdi:home` 等 `集合:名称` 标识。
   - 说明首次缺少本地数据时可能请求 Iconify API，离线环境不能依赖这种方式。
4. 离线按需组件区：
   - 演示 `<i-mdi-home />`。
   - 说明图标在构建期由 `unplugin-icons` 和 `IconsResolver` 解析。
   - 说明组件名应静态写入模板，不适合运行时拼接。
5. 自定义图标区：
   - 单色示例使用 `custom:box`，演示通过 `color` 或继承 `currentColor` 改色。
   - 多色示例使用 `custom:apple`、`custom:spring` 或 `custom:logo`，说明保留 SVG 内置颜色。
   - 说明 `src/core/svg-import.ts` 在应用启动时注册两份 JSON。
6. 自定义图标生成流程区：
   - 单色与多色 SVG 的输入目录。
   - `pnpm run icons:generate` 命令。
   - 两份 JSON 输出文件。
   - 新增 SVG 后重新生成并检查页面的步骤。
7. 常用控制区：
   - `width`、`height`、`color`。
   - `<Icon :icon="iconName" />` 动态名称示例。
   - 使用 CSS `currentColor` 继承文本颜色的示例。

代码示例直接使用 `<pre><code>` 展示，避免引入高亮依赖。实际渲染示例和代码文本放在同一区块，便于对照验证。

### 实现依赖

本组件文档会引用 `pnpm run icons:generate`。开始实现前，目标分支必须包含
`codex/svg-iconify-generation` 的实现提交 `966d8f5`，或者已经包含等价的生成命令，避免页面说明尚不存在的工作流。

## 接口兼容性

- `/icon` 路由名称、路径和 `App.vue` 导航保持不变。
- 不改变 `IconView.vue` 的对外 props 或事件；当前页面没有这些公共接口。
- `IconUsageGuide.vue` 是内部展示组件，首版不提供公共 props 或事件。
- 保留现有三种图标加载机制及 `custom` prefix。
- 显式导入新组件，避免依赖自动生成的 `components.d.ts`，不手工修改自动生成声明。

## 风险

- 在线示例在无网络环境下可能暂时不显示，需要在页面中明确标记其网络依赖，不能让它影响其他示例。
- 多色 SVG 内部若使用 `currentColor`，其颜色行为可能不同于完全固定色值的图标；说明应使用“通常保留内置颜色”，避免绝对化。
- 页面中的代码文本可能与实际示例漂移，实现时应让每个代码块紧邻对应的真实渲染示例并逐项核对。
- 新组件包含较多说明和样式，需要处理窄屏布局、代码块横向滚动和颜色对比度。
- 当前待合并的 SVG 生成分支与本计划都会修改 README；实施前应先确定提交整合顺序，遇到 Git 冲突必须停止并请求人工处理。

## 验收与验证

- 访问 `/icon` 能看到结构完整的图标使用指南。
- 在线 Iconify、离线按需组件、自定义单色和自定义多色四类示例均有实际渲染结果与代码说明。
- 单色 `custom:box` 能通过 `color` 或父元素颜色改变颜色。
- 多色图标能保持 SVG 自带颜色，页面说明不误导为所有自定义图标都可整体改色。
- 动态 `<Icon :icon="...">` 示例能切换或展示运行时图标名称。
- 页面明确列出 SVG 输入目录、JSON 输出文件、生成命令和注册文件。
- 在常见桌面宽度和窄屏下手工检查布局，代码块不撑破页面。
- 执行 `pnpm run type-check`。
- 执行 `pnpm run lint`。
- 执行 `pnpm run build-only`。
