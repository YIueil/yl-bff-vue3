# Ant Design Vue 风格文档布局优化计划

> 状态：已完成
>
> 本文档遵循 `AGENTS.md` 的“计划先行”约束。计划阶段只新增本文并更新
> `TODO.md` 链接，不修改运行时代码。

## 1. 目标与范围

参考 Ant Design Vue 官方
[Pagination 文档页](https://www.antdv.com/components/pagination-cn)的页面结构和阅读体验，
优化 `src/layouts/DocLayout.vue`，使现有组件文档站在桌面端和移动端都具备清晰、稳定的导航与
正文阅读区域。

本次范围：

- 调整文档站顶栏、左侧组件导航、正文容器和右侧页内目录的布局与视觉。
- 优化桌面端三栏宽度、滚动边界、留白、分隔线、选中态和悬停态。
- 优化窄屏下的导航折叠与正文空间，不让收起后的 Sider 遮挡正文。
- 保留现有路由、文档子页、Anchor 数据采集和滚动行为。

非目标：

- 不复制官方站点的广告、搜索、版本切换、语言切换、GitHub 链接等站点级功能。
- 不重写 `src/views/components/` 下的文档内容和示例卡片。
- 不修改公共 API、路由路径、依赖、构建配置或 Ant Design Vue 主题系统。
- 不追求对官方页面的逐像素复制；项目品牌和现有功能入口仍需保留。

## 2. 现状问题

当前 `DocLayout.vue` 已包含 Header、Sider、Content 和右侧 Anchor，但与参考页相比存在以下问题：

- 渐变顶栏视觉权重较高，当前章节标题居中，挤压品牌和“回到主站”入口；官方页采用白色、
  低干扰的固定顶栏。
- 左侧导航使用圆点、描述和大面积圆角选中背景，单项高度偏大，不利于组件数量增加后的浏览。
- 主体使用灰色背景，正文两侧留白和三栏边界不够清楚；官方页以白色阅读面为主，通过细分隔线
  建立层级。
- `.doc-content` 和内部 `.doc-scroll` 的滚动职责不够明确，`scrollContainer` 引用未参与
  Anchor 的容器配置。
- 992px 以下仅将 Sider 改为 `position: fixed`，收起触发器及遮挡关系依赖组件默认行为；
  小屏下仍可能占用或覆盖正文。
- `siderCollapsed` 同时承担用户折叠状态和窗口响应状态，窗口跨越断点时可能覆盖用户选择。
- 右侧目录宽度和缩进偏大，和正文的视觉联系较弱；窄桌面端直接隐藏后缺少替代入口。
- 存在未使用的 `.doc-affix` 样式，说明布局结构和样式实现已经发生偏差。

## 3. 推荐方案

### 3.1 页面结构

保留现有 `a-layout` 组件树，但按官方页面的阅读结构重新分配职责：

```text
固定白色 Header（品牌 / 当前栏目 / 主站入口）
└─ Body
   ├─ 固定宽度 Sider（组件分组导航，自身滚动）
   ├─ Content（唯一的主内容滚动容器）
   │  └─ 居中正文，限制最大阅读宽度
   └─ Toc（页内目录，随主体固定并自身溢出滚动）
```

- Header 高度统一为 64px，白底、下边框和轻量阴影；品牌保留 `YL` 标识与项目名。
- Sider 桌面宽度约 240px，导航改为紧凑的文本列表；选中项使用品牌蓝文字和右侧/左侧强调线，
  不再显示圆点与长描述。
- Content 采用白色背景；正文内容设置合理的最大宽度和水平留白，宽屏时不无限拉伸。
- Toc 宽度约 160px，以细竖线和蓝色激活项表达当前章节，减少容器内边距。
- Body 固定为视口剩余高度，各栏独立处理必要的纵向溢出，避免页面和内容区形成双滚动条。

### 3.2 顶栏与导航

- 顶栏左侧使用现有品牌链接，中部不再强制居中章节标题；标题作为弱化的栏目说明紧邻品牌或在
  空间不足时隐藏。
- “回到主站”改为轻量文字按钮，提供清晰的 hover 和 `:focus-visible` 状态。
- 组件导航保留 `navItems` 作为唯一数据源，继续使用 `RouterLink` 的 active class。
- 首页入口与“组件”分组保持语义分隔；描述字段先保留在类型和数据中，视图不再渲染，避免本次
  无关的数据接口调整。
- Sider 的折叠触发器使用明确的可访问名称；桌面端允许手动折叠，移动端采用抽屉式展开或完全
  隐藏，不以窄条覆盖正文。

### 3.3 响应式策略

- `>= 1200px`：展示完整左侧导航、居中正文和右侧目录。
- `992px–1199px`：隐藏右侧目录，将空间让给正文；左侧导航保持可用。
- `< 992px`：左侧导航退出普通文档流，由 Header 中的菜单按钮控制临时导航面板；正文占满宽度。
- `< 640px`：隐藏非必要品牌文字，缩小正文水平留白，确保标题、表格和示例区不会因 Layout
  固定宽度产生额外横向滚动。

窗口断点状态与用户操作状态分离，避免 resize 后出现“视觉已隐藏但组件仍占位”或用户折叠选择被
无条件覆盖的问题。

### 3.4 Anchor 与滚动

- 保留路由变化后通过 `data-anchor` 收集目录项的机制。
- 明确 `.doc-content` 为 Anchor 的滚动容器，并通过组件支持的容器配置或等价方式让激活计算与
  实际滚动容器一致。
- 点击目录继续平滑滚动并更新 URL hash，但使用路由友好的 history 更新方式，避免破坏当前
  `/components/*` 路径和浏览器前进后退行为。
- 章节目标统一设置 `scroll-margin-top`；若 Header 不在内容滚动容器内，则偏移值按实际结构
  校准，不机械沿用 64px。
- 兼容无目录页面，保留“本页无目录”的轻量空状态。

### 3.5 样式与可访问性

- 颜色收敛为 Ant Design Vue 接近的中性色、`#1677ff` 品牌蓝、`#f0f0f0` 分隔线和白色背景。
- 交互元素补齐 `:focus-visible`，不能只靠颜色表达菜单当前状态。
- 菜单按钮维护 `aria-expanded`，导航面板保持 `aria-label="组件导航"`。
- 尊重 `prefers-reduced-motion`，减少平滑滚动或过渡动画。
- 继续使用 `<style scoped>`，只通过必要的 `:deep()` 覆盖 Ant Design Vue 内部 Anchor/Sider
  结构，不新增全局 CSS 污染。

## 4. 接口兼容性

- `DocLayout.vue` 的路由出口、`navItems` 路径、`route.meta.title` 和文档页 `data-anchor`
  约定保持不变。
- 不改变 `src/router/index.ts` 的路径、名称或懒加载方式。
- 不改变任何组件公共 props、事件、类型声明或依赖版本。
- URL hash 仍可用于章节定位；实现时需验证刷新带 hash 的页面和浏览器前进/后退。
- 视觉样式会发生明显变化，但不构成代码接口破坏。

## 5. 风险与控制

- **Anchor 激活错误**：自定义滚动容器若未正确传递给 Anchor，会导致高亮不同步。通过点击、
  滚动、刷新带 hash 三种路径验证。
- **移动端遮挡**：临时导航若层级、遮罩或关闭时机处理错误会挡住正文。验证菜单按钮、路由切换、
  遮罩关闭和窗口放大后的状态复位。
- **三栏宽度回归**：文档子页含表格、代码块和 ECharts；限制正文宽度时必须确保内部内容可滚动
  或自适应，而不是裁切。
- **样式穿透脆弱**：尽量依赖公开 class 和组件 props；深层选择器限制在 Anchor 必需范围。
- **官方页面变化**：参考页只作为当前视觉与信息架构基准，不在运行时依赖官方资源。

## 6. 验收与验证

自动校验：

- `pnpm run type-check`
- `pnpm run lint`
- `pnpm run test:run`
- `pnpm run build-only`

手工验收：

- 在 `/components/icon`、`/components/modal`、`/components/echarts` 检查长内容、复杂示例和图表。
- 桌面端 1440px：Header、左侧导航、正文和右侧目录边界清晰，正文阅读宽度稳定。
- 1024px：右侧目录按策略隐藏，左侧导航与正文均不重叠。
- 768px 和 375px：导航可打开、关闭和切换路由，正文无 Layout 引起的横向滚动或遮挡。
- 当前路由菜单状态正确；目录点击、滚动高亮、hash 更新和浏览器前进/后退正确。
- 键盘可聚焦品牌、主站入口、菜单按钮、左侧路由和右侧目录；焦点轮廓可见。
- `prefers-reduced-motion` 下不强制执行明显的平滑动画。

1. 开发者确认并在当前主分支提交本文及 `TODO.md` 链接。

3. 在 Worktree 中重构 `DocLayout.vue` 的状态、模板和 scoped 样式。

5. 运行自动校验并完成上述手工验收。
6. 验收通过后更新 `TODO.md` 状态，使用临时 Codex 身份创建 Conventional Commit。
7. 输出修改文件、核心逻辑、风险点和 Diff 摘要；不自动合并。
